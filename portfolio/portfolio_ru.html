Квиз на знание JavaScript

Квиз корректно отображается на всех размерах экранов (320px-1920px).

Код написан на чистом JavaScript.

В работе представлена непрерывная работа как с запросами данных с серверам, так и с отправкой данных на него.

Стоит выделить следующие моменты, которые присутствуют в работе:
● форма регистрации пользователя проходит валидацию с помощью регулярных выражений;
● введенные пользователем данные во время регистрации сохраняются в SessionStorage;
● все динамические данные (названия тестов, вопросы в тестах, варианты ответов и пр.) созданы посредством JavaScript, как DOM-элементы;
● содержание элементов взято из сервера, путем GET-запросов;
● настроена логика для перемещения между вопросами (назад/дальше/пропустить/завершить);
● ответы пользователя сохраняются;
● результат генерируется исходя из выбранных пользователем идентификаторов ответов;
● правильные/ неправильные ответы показываются пользователю по итогам сравнения выбранных им идентификаторов ответов с верными ответами на тесте, которые получаются GET-запросом с сервера.

Онлайн-заказ бургеров

Сайт адаптирован под размеры экрана 570px-1920px.

Проект выполнен с помощью фреймворка Angular в рамках интенсива онлайн-школы «Айтилогия».

● при загрузке страницы появляется анимированный лоадер в виде бургера;
● по умолчанию валюта прайса — доллар, но если в шапке страницы нажать на него, то валюта поменяется, конвертация произойдет автоматически;

● первое поле запрещено редактировать (изменить значение можно только выбором необходимого бургера и нажатием на его карточке на 'в корзину');

● если какое-либо поле не заполнено, то рамка подсвечивается красным цветом.

Форма регистрации и логина

Форма доступна для просмотра и использования на всех размеров экранов.

Функционал страницы выполнен на чистом JavaScript.

Валидация формы реализована посредством регулярных выражений. У пользователя нет шансов заполнить форму неправильно. При несоответствующем заполнении поля появляется предупрждение о том, что именно пользователь вводит не так.

Ключевые моменты, настроенные с помощью регулярных выражений:
● все поля должны быть заполнены;
● поле 'Full Name' может содержать только буквы и пробел;
● поле 'Your username' может содержать только буквы, цифры, символ подчеркивания и тире;
● введенный e-mail также проверяется на корректность;
● поле пароля должно содержать минимум 8 символов, среди которых есть:
    - хотя бы одна буква в верхнем регистре;
    - хотя бы одна цифра;
    - хотя бы один спецсимвол;
● 'Password' и 'Repeat Password' должны совпадать;
● пользователь обязательно должен согласиться с условиями.

Также в этой работе оттачивался навык работы с Local Storage: информация о пользователе записывается в объект, а далее в массив. Это позволяет проверить введенные при входе в систему данные на соответствие с теми, которые пользователь использовал при регистрации. Не зарегистрировался — не залогинился.
